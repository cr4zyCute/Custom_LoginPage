# White Label Authentication System (Multi-Stack)

This project is a comprehensive, white-label authentication solution designed for resale. It offers multiple technology stack variants to cater to different buyer infrastructure requirements.

## 🏗️ Multi-Stack Strategy

The project is organized into 5 variants, each implementing the same high-quality UI/UX and feature set but using different backend technologies.

### 📁 Variants

1.  **[Variant 1: Next.js + Prisma + PostgreSQL](./Variant_1_NextJS_Prisma_Postgres)** (✅ Implemented)
    -   **Stack:** Next.js 14, Prisma ORM, PostgreSQL (SQLite for dev), NextAuth.js.
    -   **Status:** Complete. Includes Theme System, Auth Pages/Modals, and Database Seeding.

2.  **Variant 2: Next.js + Mongoose + MongoDB** (📋 Planned)
    -   **Stack:** Next.js, Mongoose ODM, MongoDB.
    -   **Status:** Folder created.

3.  **Variant 3: Next.js + Drizzle + MySQL** (📋 Planned)
    -   **Stack:** Next.js, Drizzle ORM, MySQL.
    -   **Status:** Folder created.

4.  **Variant 4: Next.js + Supabase** (📋 Planned)
    -   **Stack:** Next.js, Supabase (Auth & DB).
    -   **Status:** Folder created.

5.  **Variant 5: Next.js + Firebase** (📋 Planned)
    -   **Stack:** Next.js, Firebase (Auth & Firestore).
    -   **Status:** Folder created.

## 🌟 Core Features (All Variants)

-   **26+ Themes:** Extensive theme library (Modern, Retro, Corporate, etc.).
-   **Authentication:** Login, Register, OAuth, OTP (planned).
-   **White Label:** Fully customizable branding and styles.
-   **Responsive Design:** Mobile-first UI with Shadcn UI and Tailwind CSS.

## 🚀 Getting Started

To start working with the primary variant:

```bash
cd Variant_1_NextJS_Prisma_Postgres
npm install
npm run dev
```

Refer to the README within each variant folder for specific setup instructions.
